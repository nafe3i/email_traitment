  1. users          → Enregistrement des utilisateurs

  3. rag_documents  → Base de connaissances

🎯 SCHÉMA PHASE 2 EN BREF:

TABLE 1: users
  Colonnes: id, email, hashed_password, role, is_active, is_verified, 
            two_factor_enabled, created_at, updated_at
  Capacité: Utilisateurs illimités
  Purpose: Enregistrement des utilisateurs

TABLE 2: email_logs
  Colonnes: id, user_id, sender, subject, body, response, status, 
            response_time_ms, model_used, tokens_used, cost_cents
  Capacité: Millions d'emails
  Purpose: Historique complet de traitement

TABLE 3: rag_documents
  Colonnes: id, user_id, chroma_id, title, content, category, tags, 
            embedding_model, created_at
  Capacité: Milliers de documents
  Purpose: Base de connaissances pour RAG

TABLE 4: api_tokens
  Colonnes: id, user_id, token_hash, scopes, expires_at, is_active, created_at
  Capacité: Tokens illimités par utilisateur
  Purpose: Gestion d'accès programmatique

TABLE 5: audit_logs
  Colonnes: id, user_id, ip_address, action, resource, status, created_at
  Capacité: Milliards d'événements
  Purpose: Traçage de sécurité complet
